# 进销存体系合并方案

## 现状问题

| 系统 | 位置 | 数据表 | 规格格式 |
|------|------|--------|---------|
| **InventoryManage** | 库存与利润页面 | `purchase_records`, `sales_records` | `volume_ml` 纯数字（如 `30`） |
| **ProductList** | 产品编辑 → 进销存 | `products` 表累计字段 | `size` 带单位（如 `30ml`, `1个`） |

两套系统互不感知，数据割裂。

## 合并方案

### 数据层（数据库）

**`purchase_records` 表** 增加一个字段：
- `size_label` (`text`): 规格标签，如 `"30ml"`, `"1个"`, `"1套"`
- `unit_type` (`text`): 单位类型，`"ml"` 或 `"piece"`

**`sales_records` 表** 同样增加：
- `size_label` (`text`): 规格标签
- `unit_type` (`text`): 单位类型

### 核心逻辑变更

1. **唯一数据源** → `purchase_records` + `sales_records` 表是所有进销存操作的真实数据源
2. **所有录入操作都写这俩表**，不分是在哪操作
3. **products 表的累计字段** → 改为数据库函数实时计算，或每次做 CRUD 时同步更新

### 规格处理

- **元/和/生系列** → 单位 `ml`，规格可选 5ml/10ml/15ml/30ml/50ml/100ml/500ml/1000ml
- **香(境)系列** → 单位 `piece`，规格可选 1个/1件/1套/5个/10件
- **InventoryManage** 的进货下拉规格列表与 ProductList 同步
- 前端录入时存 `size_label`（如 `30ml` → volume_ml=30, size_label="30ml", unit_type="ml"）

### 修改范围

1. `database.types.ts` — 接口增加 `size_label` 和 `unit_type`
2. `dataService.ts` — `purchaseService` / `salesService` 的 create 处理规格字段
3. `ProductList.tsx` — 入库/出库保存改为同时写 `purchase_records` / `sales_records`
4. `InventoryManage.tsx` — 进货/销售表单增加规格下拉选择
5. `inventoryService` — 实时汇总逻辑不变，数据源统一
